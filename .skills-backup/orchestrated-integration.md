# orchestrated-integration

## 目标

前后端联调，验证完整流程，产出最终交付物。

## 前置条件

verification 阶段已通过（所有 🔴 阻塞问题已解决）。

## 输入

- architecture 阶段的设计文档
- implementation 阶段各职能的交付物
- verification 阶段的测试报告

## 执行流程

### 1. 联调准备

```markdown
## 联调准备

### 已就绪组件
- [ ] 后端服务：状态 + 部署地址
- [ ] 前端应用：状态 + 访问地址
- [ ] 数据库：连接信息
- [ ] 测试环境：是否可用

### 联调用例
| # | 场景 | 步骤 | 预期结果 |
|---|------|------|---------|
| 1 | 用户注册流程 | 前端表单 → 后端 API → 数据库 | 注册成功，收到验证邮件 |
| 2 | 用户登录流程 | 前端登录 → 后端认证 → 返回 Token | 登录成功，获取 JWT |
| ... | ... | ... | ... |
```

### 2. 端到端联调

按联调用例逐项执行：

```markdown
## 联调执行

### 联调用例 #1：用户注册流程
- **状态**：✅ 通过 / ❌ 失败
- **执行截图**：[截图链接]
- **实际行为**：[描述]
- **与预期对比**：[是否一致]

### 联调用例 #2：用户登录流程
- **状态**：✅ 通过 / ❌ 失败
- **执行截图**：[截图链接]
- **实际行为**：[描述]
- **与预期对比**：[是否一致]
```

### 3. 部署准备（如需要）

```markdown
## 部署指南

### 环境要求
- Node.js 版本：
- 数据库版本：
- 其他依赖：

3. [步骤]

|--------|------|-------|
| DATABASE_URL | 数据库连接 | postgresql://... |
| JWT_SECRET | JWT 密钥 | [生产环境需重新生成] |

### 验证部署
[部署后如何验证系统正常]
```

### 4. 文档整理

整理所有交付物：

```markdown
## 交付文档

### 用户手册
[终端用户如何使用这个功能]

### 开发文档
[开发者如何接入和维护]

[运维人员如何部署和监控]
```

## 最终交付判定

**最终判定**：✅ 交付完成 / ⚠️ 有遗留问题

- **交付完成**：联调通过 + 文档齐全 + 验收确认
- **有遗留问题**：记录未解决问题 + 说明原因 + 给出处理建议
